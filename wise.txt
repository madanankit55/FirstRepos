	,[USD_K$] = 
	CAST(ROUND(SUM(A.GROUP_CURRENCY) * (CASE WHEN min(B.SortP2) = 'P2-01' THEN -1 ELSE 1 END)  / 1000,0) AS INT)  
	,[%] =
	CASE WHEN 
	SUM(A.GROUP_CURRENCY) * (CASE WHEN min(B.SortP2) = 'P2-01' THEN -1 ELSE 1 END) / 
	(SELECT COUNT(*) FROM JCPCOPA.Dim_NIS_Hierarchy WHERE P2 = 'Total Revenue') > 1 
	THEN 
	CONCAT(CAST(ROUND(1 * 100, 0) as INT) , '%')
 
	WHEN 
	SUM(A.GROUP_CURRENCY) * (CASE WHEN MIN(B.SortP2) = 'P2-01' THEN -1 ELSE 1 END) / 
	(SELECT COUNT(*) FROM JCPCOPA.Dim_NIS_Hierarchy WHERE P2 = 'Total Revenue') < -1 
	THEN 
	CONCAT(CAST(ROUND(-1 * 100, 0) as INT)  , '%')
 
	ELSE 
	CONCAT(CAST(
	(SELECT COUNT(*) FROM JCPCOPA.Dim_NIS_Hierarchy WHERE P2 = 'Total Revenue') * 100.0 / 
	(SELECT SUM(COUNT(*)) OVER() FROM JCPCOPA.Dim_NIS_Hierarchy WHERE P2 = 'Total Revenue')

Azure Synapse Analytics

Dedicated SQL Pool support three methods for distributing data.

i) Hash Distribution: Hash Distribution tables, distributed tables rows across the compute node by using hash function to assign each row to one distribution.
Improves query peformance on large fact tables.
Where to use hash distribution table when we have an table disk size is more than 2 GB and table is frequently insert, update and delete operations.

Ex - 
CREATE TABLE [dbo].[FactInternetSales]
(   [ProductKey]            int          NOT NULL
,   [OrderDateKey]          int          NOT NULL
,   [CustomerKey]           int          NOT NULL
,   [PromotionKey]          int          NOT NULL
,   [SalesOrderNumber]      nvarchar(20) NOT NULL
,   [OrderQuantity]         smallint     NOT NULL
,   [UnitPrice]             money        NOT NULL
,   [SalesAmount]           money        NOT NULL
)
WITH
(   CLUSTERED COLUMNSTORE INDEX
,  DISTRIBUTION = HASH([ProductKey],   [OrderDateKey],   [CustomerKey] ,   [PromotionKey])
);

ii) Round Robin Distribution: Round Robin distribution tables, distributes tables rows evenly across distribution, useful for improving loading speed.

Ex -
CREATE TABLE [dbo].[HackneyLicense]
(
    [HackneyLicenseID] int NOT NULL,
    [HackneyLicenseBKey] varchar(50)  NOT NULL,
    [HackneyLicenseCode] varchar(50)  NULL
)
WITH
(
    DISTRIBUTION = ROUND_ROBIN,
    CLUSTERED COLUMNSTORE INDEX
);

iii) Replicated: replicated table has a full copy of the table available on every compute node, queries run fast on replicated.

ROW_NUMBER() OVER (PARTITION By employee order by employee,dates) as 'rn'
